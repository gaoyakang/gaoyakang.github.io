产品的诞生

一款互联网产品要在合理的设计与规划下才能有条不紊的诞生，这个设计与规划的过程叫做软件工程管理，现在常见的互联网产品有:

  • ①操作系统(linux,MacOS,Windows)
  • ②桌面应用(通讯/文档/游戏/娱乐)
  • ③网站应用(PC端/Mobile端/微信端)
  • ④移动应用(通讯/文档/游戏/娱乐)

这些产品形态各异，不仅针对的服务人群不同，而且开发维护时所采用的技术栈也不相同，为了保质保量的完成任务，就必须要遵循以下开发流程: 分析-设计-开发-测试-部署-运维，由此衍生出了不同的互联网工种，下面以一个实际项目的开发过程来说明:

  • 1.公司收到项目任务
  • 2.PM需求分析，画原型图
  • 3.UI设计交互图和效果图
  • 4.Coder组长拟定并评估开发方案后执行
  • 5.QA进行功能测试，视图测试和压力测试
  • 6.OP部署上线，后期维护

以上是一个比较合理的开发流程，但受限于现实种种情况(资金/意识/习惯)，有时候我们并不能完全按照这样的流程来进行开发，写代码的Coder可能要动手弄设计，测试和运维也可能由同一队人负责，但这个流程再缩减也必须保证开发-测试-部署这三环，利用敏捷开发快速实现产品原型，后期再进行视图与性能优化，通过迭代的方式上线新功能，相对而言，公司开发流程的完善程度在一定程度上代表着公司的实力。接下来以一个简易在线商城Mall为例来体验一下开发整体流程，注意有些技术不会不要紧，关键是要坚持做完，感受各工种差异。

Mall需求分析

在需求分析的阶段，PM(项目经理)会召集设计，开发，测试，运维一起来进行讨论，大家对产品经理提出的需求进行判断，合理的提出质疑，明确自己能够实现的功能，这个过程中最难的部分是要大家建立共同的思维标准，各人明确任务，避免后续返工，拿Mall来分析，首先功能主线要确保无误，在此基础上需要留好扩展的余地，为后期迭代做准备，一个在线商城具体的主线功能如下:

  • ①访问首页，展示随机商品列表
  • ②向下拖拽，列表懒加载
  • ③设置条件进行筛选
  • ③点击列表中任意一个跳转详情页
  • ④详情页中可以查看详情
  • ⑤加入购物车前需要登录
  • ⑥登陆后可添加商品到购物车
  • ⑦添加购物车后可以去结算
  • ⑧结算时需要添加收货地址
  • ⑨添加好收货地址后可以再次查看购物车
  • ⑩确认购物车后可以前去支付
  • ①支付完成后可以生成订单

在确定主线功能时，需要以用户的角度来思考问题，总结这些步骤可以发现这其中主要包括用户模块，商品模块，用户可以浏览商品，登录，加购物车，加地址，付款，并且这些操作还伴随着权限，为了实现主线快速功能上线，我们将商品详情与用户支付部分弱化，来进行下一步的开发，经过讨论要达到以下目的:

  • ①设计知道自己要做哪些交互和视图
  • ②开发知道自己要实现哪些功能
  • ③测试确定测试时间
  • ④运维协助大家完成任务

  • 2.tar -xzvf解压
  • 3.mv到/usr/local/xx

(二)搭建mongodb环境

  • 1.wget <node官网linux版mongodb下载链接>
  • 2.tar -xzvf解压
  • 3.mv 解压文件到/usr/local/xx
  • 4.ln -s /usr/local/xx/bin/xx /usr/local/bin
  • 5.执行mongod和mongo, use 库名
  • 6.mongoimport -d 库 -c表 –file 地址

(三)下载git拉取源码

  • 1.安装git

yum install git

  • 2.配置部署公钥

①服务器: ssh-keygen -t rsa -C "1@qq.com"
②服务器: cat ～/.ssh/id_rsa.pub
③github: 将公钥放在部署公钥中

  • 3.拉取源码

①mkdir deploy
②git clone <仓库ssh地址>
③npm install(这里是后端代码)
④node启动服务(保证数据库已经启动)

(四)常驻服务并转发

  • 4.常驻服务器
    ①npm install pm2 -g
    ②ln -s /usr/local/node/bin/pm2 /usr/local/bin
    ③配置安全组端口(0.0.0.0/0)
    ④pm2 srart mongod和www
    ⑤外网ip访问

  • 5.编译前端源码并实现跨域
    ①npm run build
    ②搭建nginx服务器:https://m.runoob.com/linux/nginx-install-setup.html(到查看版本即可)
    ③启动nginx -c /usr/local/webserver/nginx/conf/nginx.conf
    ④配置安全组80端口(可以直接访问ip看到nginx欢迎页)
    ⑤npm run build打包源码
    ⑥mv dist /usr/local/webserver/

⑦vim nginx.conf中将location:

在域名解析的地方给它添加一个cname对应到服务器ip后等待十来分钟，如果要上线网站，需要对网站进行备案，否则就无法访问，备案好的网站就能被访问到了，后续还可以申请SSL证书保证安全。
